Le problème qu'il résout

Les assistants IA de code (Claude Code, Cursor, Copilot…) lisent les fichiers un par un, à la demande.

  • Ils lisent des dizaines de fichiers pour suivre une seule chaîne d'appels.
  • Ils repartent de zéro à chaque conversation.
  • Ils remplissent la fenêtre de contexte de code brut, sans place pour réfléchir.
  • Et une recherche texte ne peut pas calculer un rayon d'impact transitif — grep trouve où un nom apparaît, pas la chaîne de choses qui cassent trois sauts plus loin.

Code Explorer pré-calcule tout ça, une fois, dans un graphe qui vit sur le disque et répond instantanément aux questions de structure.

Ce que le graphe apporte à ton agent

🕸️

Graphe de connaissance pré-indexé

Tout ton dépôt parsé en symboles et relations typées — fonctions, classes, appels, imports, hiérarchie. Persiste sur le disque, interrogeable en un appel.

Démarrage rapide

# 1. Build (release, binaire ~35 Mo)
cargo build --release        # binaire dans target/release/gitnexus

# 2. Indexer le dépôt une fois (relancer avec --incremental après changements)
gitnexus analyze .

# 3. Brancher ton agent dessus via MCP
gitnexus mcp-install         # écrit .mcp.json pour Claude Code
# …ou à la main :  { "command": "gitnexus", "args": ["mcp"] }

Vérif : ton agent doit afficher « gitnexus · 30 tools ».
